You are building `ft-nixpalette-niri` — a NixOS/Home Manager flake that integrates
the ft-nixpalette theme framework with the niri scrollable-tiling Wayland compositor.

## What ft-nixpalette provides (the dependency)

ft-nixpalette (github:FT-nixforge/ft-nixpalette) is a theme management flake built on
top of Stylix. At build time it writes two files:

  NixOS:        /etc/ft-nixpalette/colors.json
                /etc/ft-nixpalette/themes.json
  Home Manager: $XDG_DATA_HOME/ft-nixpalette/colors.json
                $XDG_DATA_HOME/ft-nixpalette/themes.json

colors.json structure:
  { "themeId": "builtin:base/catppuccin-mocha", "polarity": "dark", "base16": { "base00": "1e1e2e", ... } }

themes.json structure:
  { "<themeId>": { "themeId": "...", "polarity": "...", "base16": { ... }, "meta": { ... } }, ... }

All base16 hex values are plain 6-character strings WITHOUT a leading #.
Standard semantic mapping: base00=background, base05=foreground, base08=red,
base0B=green, base0C=cyan, base0D=blue, base0E=purple.

## What ft-nixpalette-niri should provide

A separate flake with nixosModules.default and homeModules.default that
adds niri-ecosystem theming on top of ft-nixpalette. All options live under the
`ft-nixpalette-niri` namespace.

niri uses a KDL-based configuration file (niri/config.kdl). Home Manager support
for niri is provided by the niri-flake (github:sodiboo/niri-flake) which exposes
programs.niri.settings as a Nix attrset. Prefer the attrset approach over raw KDL
string injection where possible.

### 1. swww — wallpaper (niri does not have a built-in wallpaper setter)
- Declare `ft-nixpalette-niri.swww.enable` (default: true)
- On niri startup (spawn-at-startup in config) run swww-daemon and swww img
- Provide `ft-nixpalette-niri.swww.wallpaper` override option (null or path)
- Support static and animated (GIF) wallpapers — swww handles both

### 2. swaylock — lockscreen (works on niri via swaylock-effects or just swaylock)
- Declare `ft-nixpalette-niri.swaylock.enable` (default: true)
- Generate swaylock config from base16 colors (same mapping as ft-nixpalette-sway):
    inside-color  → base00
    ring-color    → base0D
    ring-ver-color → base0B
    ring-wrong-color → base08
    text-color    → base05
    line-color    → base03

### 3. waybar
- Declare `ft-nixpalette-niri.waybar.enable` (default: false)
- Inject CSS @define-color variables for all 16 base16 slots
- niri-specific: waybar needs `layer-shell` output; document the required
  waybar config for niri (use `sway/workspaces` module with niri IPC)

### 4. mako — notification daemon
- Declare `ft-nixpalette-niri.mako.enable` (default: true)
- Same color mapping as ft-nixpalette-sway

### 5. niri border and focus colors
- Declare `ft-nixpalette-niri.borders.enable` (default: true)
- Set via programs.niri.settings.layout.border:
    active-color   → base0D (blue)
    inactive-color → base03 (comments/dim)
- Set programs.niri.settings.layout.focus-ring:
    active-color   → base0E (purple — focus ring is distinct from border in niri)
    inactive-color → base02

### 6. fuzzel — niri's natural launcher companion
- Declare `ft-nixpalette-niri.fuzzel.enable` (default: true)
- Generate programs.fuzzel.settings colors from base16:
    background → base00 + "ff" (add alpha)
    text       → base05 + "ff"
    match      → base0D + "ff"
    selection  → base02 + "ff"
    selection-text → base05 + "ff"
    border     → base0D + "ff"

### 7. Live theme switcher script
- Provide package `ft-nixpalette-niri.packages.${system}.ft-nixpalette-switch`
- Script reads themes.json, presents fzf or fuzzel picker, then on selection:
    1. swww img <wallpaper>
    2. Writes ~/.cache/ft-nixpalette/active.json
    3. Reloads waybar (pkill -SIGUSR2 waybar)
    4. makoctl reload
    5. notify-send confirmation
- Works only for themes in themes.json (ft-nixpalette.preloadThemes)

## Architecture requirements

- flake inputs:
    inputs."ft-nixpalette".url = "github:FT-nixforge/ft-nixpalette"
    inputs.niri-flake.url = "github:sodiboo/niri-flake"  # for HM programs.niri
- Re-export ft-nixpalette's modules; users import only ft-nixpalette-niri
- Do NOT re-declare ft-nixpalette options
- All features default to true for core (swww, swaylock, mako, borders, fuzzel),
  false for potentially conflicting (waybar)

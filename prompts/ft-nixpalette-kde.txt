You are building `ft-nixpalette-kde` — a NixOS/Home Manager flake that integrates
the ft-nixpalette theme framework with the KDE Plasma 6 desktop environment.

## What ft-nixpalette provides (the dependency)

ft-nixpalette (github:FT-nixforge/ft-nixpalette) is a theme management flake built on
top of Stylix. At build time it writes two files:

  NixOS:        /etc/ft-nixpalette/colors.json
                /etc/ft-nixpalette/themes.json
  Home Manager: $XDG_DATA_HOME/ft-nixpalette/colors.json
                $XDG_DATA_HOME/ft-nixpalette/themes.json

colors.json structure:
  { "themeId": "builtin:base/nord", "polarity": "dark", "base16": { "base00": "2e3440", ... } }

themes.json structure:
  { "<themeId>": { "themeId": "...", "polarity": "...", "base16": { ... }, "meta": { ... } }, ... }

All base16 hex values are plain 6-character strings WITHOUT a leading #.
Standard semantic mapping: base00=background, base05=foreground, base08=red,
base0B=green, base0D=blue, base0E=purple.
`polarity` ("dark"/"light") maps to KDE's color scheme mode.

## What ft-nixpalette-kde should provide

A separate flake with nixosModules.default and homeModules.default that
adds KDE Plasma 6 theming on top of ft-nixpalette. All options live under the
`ft-nixpalette-kde` namespace.

KDE Plasma theming is done primarily via:
- plasma-manager (Home Manager module for Plasma 6, github:nix-community/plasma-manager)
  which exposes programs.plasma6.* options
- KDE color scheme files (.colors) written to ~/.local/share/color-schemes/
- plasma-apply-colorscheme CLI for runtime application

Prefer plasma-manager's attrset approach over raw config file manipulation.

### 1. Wallpaper via plasma-manager
- Declare `ft-nixpalette-kde.wallpaper.enable` (default: true)
- Set programs.plasma6.workspace.wallpaper to the wallpaper path from
  ft-nixpalette.defaultWallpaper or the active theme
- Also support slideshow and color wallpaper types as fallback

### 2. KDE Color Scheme generation
- Declare `ft-nixpalette-kde.colorScheme.enable` (default: true)
- Generate a .colors file in the KDE color scheme INI format and install it to
  ~/.local/share/color-schemes/ft-nixpalette-<themeId>.colors
- Map base16 → KDE color roles:
    [Colors:Window]
      BackgroundNormal  → base01
      ForegroundNormal  → base05
    [Colors:View]
      BackgroundNormal  → base00
      ForegroundNormal  → base05
      DecorationFocus   → base0D
      DecorationHover   → base0E
    [Colors:Button]
      BackgroundNormal  → base02
      ForegroundNormal  → base05
    [Colors:Selection]
      BackgroundNormal  → base0D
      ForegroundNormal  → base00
    [Colors:Tooltip]
      BackgroundNormal  → base01
      ForegroundNormal  → base05
    [General]
      ColorScheme       → ft-nixpalette-<themeId>
- Apply via programs.plasma6.colorScheme = "ft-nixpalette-<themeId>"
- Set dark/light mode via programs.plasma6.workspace.colorScheme or kwriteconfig6

### 3. Konsole color scheme
- Declare `ft-nixpalette-kde.konsole.enable` (default: true)
- Generate a Konsole color scheme file (.colorscheme) in
  ~/.local/share/konsole/ft-nixpalette-<themeId>.colorscheme
- Map base16 to the 16 ANSI slots + background/foreground
- Set as default profile color scheme via programs.plasma6 or direct file

### 4. KWin border colors
- Declare `ft-nixpalette-kde.kwin.enable` (default: true)
- Set KWin decoration colors through the color scheme (already handled in step 2)
- Optionally set via kwriteconfig6 for accent color:
    kdeglobals → [General] AccentColor → base0D

### 5. Live theme switcher
- Provide package `ft-nixpalette-kde.packages.${system}.ft-nixpalette-switch`
- Script reads themes.json, presents kdialog or fzf picker, then on selection:
    1. Writes the new .colors file to ~/.local/share/color-schemes/
    2. Runs plasma-apply-colorscheme ft-nixpalette-<themeId>
    3. Runs plasma-apply-desktoptheme if applicable
    4. Changes wallpaper via dbus:
         qdbus org.kde.plasmashell /PlasmaShell
           org.kde.PlasmaShell.evaluateScript "..."
    5. Writes ~/.cache/ft-nixpalette/active.json
- Works only for themes in themes.json (ft-nixpalette.preloadThemes)
- KDE has excellent runtime theming support — this switcher can apply colors,
  wallpaper, and mode without any logout

## Architecture requirements

- flake inputs:
    inputs."ft-nixpalette".url      = "github:FT-nixforge/ft-nixpalette"
    inputs.plasma-manager.url  = "github:nix-community/plasma-manager"
- Re-export ft-nixpalette's modules and plasma-manager's HM module
- Do NOT re-declare ft-nixpalette options
- Home Manager module handles color schemes, Konsole, wallpaper, kwin via plasma-manager
- NixOS module ensures plasma-apply-colorscheme and kwriteconfig6 are in PATH

You are building `nixpalette-sway` — a NixOS/Home Manager flake that integrates
the nixpalette theme framework with the Sway compositor ecosystem.

## What nixpalette provides (the dependency)

nixpalette (github:FTMahringer/nixpalette) is a theme management flake built on
top of Stylix. At build time it writes two files:

  NixOS:        /etc/nixpalette/colors.json
                /etc/nixpalette/themes.json
  Home Manager: $XDG_DATA_HOME/nixpalette/colors.json
                $XDG_DATA_HOME/nixpalette/themes.json

colors.json structure:
  { "themeId": "builtin:base/nord", "polarity": "dark", "base16": { "base00": "2e3440", ... } }

themes.json structure:
  { "<themeId>": { "themeId": "...", "polarity": "...", "base16": { ... }, "meta": { ... } }, ... }

All base16 hex values are plain 6-character strings WITHOUT a leading #.
Standard semantic mapping: base00=background, base05=foreground, base08=red,
base0B=green, base0C=cyan, base0D=blue, base0E=purple.

## What nixpalette-sway should provide

A separate flake with nixosModules.default and homeModules.default that
adds Sway-ecosystem theming on top of nixpalette. All options live under the
`nixpalette-sway` namespace.

### 1. swaybg — wallpaper
- Declare `nixpalette-sway.swaybg.enable` (default: true)
- Set the wallpaper from colors.json on session start via exec_always in sway config
- Provide `nixpalette-sway.swaybg.wallpaper` option (null or path) as override
- swaybg only supports static images; document this clearly

### 2. swaylock — lockscreen
- Declare `nixpalette-sway.swaylock.enable` (default: true)
- Generate a swaylock config using base16 colors:
    inside-color       → base00 (background)
    ring-color         → base0D (blue — idle ring)
    ring-ver-color     → base0B (green — verifying)
    ring-wrong-color   → base08 (red — wrong)
    key-hl-color       → base0D (key highlight)
    text-color         → base05 (foreground)
    line-color         → base03
    separator-color    → base01
- Use Home Manager's programs.swaylock.settings attrset

### 3. waybar
- Declare `nixpalette-sway.waybar.enable` (default: false)
- Inject CSS variables for all 16 base16 colors into waybar style:
    @define-color base00 #<base00>;
    ... all 16 slots ...
- Prepend the variable block to the user's existing waybar CSS so they can
  reference @base0D etc. without hardcoding colors

### 4. mako — notification daemon
- Declare `nixpalette-sway.mako.enable` (default: true)
- Generate mako colors from base16:
    background-color → base01
    border-color     → base0D
    text-color       → base05
    border-size      → 2
  For urgency-critical: border-color → base08

### 5. rofi / wofi
- Declare `nixpalette-sway.rofi.enable` and `nixpalette-sway.wofi.enable`
- Generate matching color themes from base16 colors

### 6. Sway border colors
- Declare `nixpalette-sway.borders.enable` (default: true)
- Inject sway `client.*` color settings via wayland.windowManager.sway.config.colors:
    focused border        → base0D
    focused background    → base0D
    focused text          → base00
    unfocused border      → base03
    unfocused background  → base01
    unfocused text        → base05
    urgent border         → base08
    urgent background     → base08
    urgent text           → base00

### 7. Live theme switcher script
- Provide package `nixpalette-sway.packages.${system}.nixpalette-switch`
- Script reads themes.json, lets the user pick from preloaded themes, then:
    1. Calls swaybg / pkill swaybg + swaybg to reload wallpaper
    2. Writes ~/.cache/nixpalette/active.json
    3. Reloads waybar (pkill -SIGUSR2 waybar)
    4. Reloads mako (makoctl reload)
    5. Optionally sends notify-send notification
- Works only for themes in themes.json (nixpalette.preloadThemes)

## Architecture requirements

- flake input: inputs.nixpalette.url = "github:FTMahringer/nixpalette"
- Re-export nixpalette's modules so users only need to import nixpalette-sway
- Do NOT re-declare nixpalette options — read from config.nixpalette.* and JSON files
- All features default to true only for core (swaybg, swaylock, mako, borders),
  false for potentially conflicting ones (waybar, rofi, wofi)
- Home Manager module handles user-space (swaylock, waybar CSS, mako, rofi/wofi,
  switcher in PATH, sway border colors)
- NixOS module handles system packages (swaybg, swaylock, mako installed)

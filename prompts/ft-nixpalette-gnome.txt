You are building `ft-nixpalette-gnome` — a NixOS/Home Manager flake that integrates
the ft-nixpalette theme framework with the GNOME desktop environment.

## What ft-nixpalette provides (the dependency)

ft-nixpalette (github:FT-nixforge/ft-nixpalette) is a theme management flake built on
top of Stylix. At build time it writes two files:

  NixOS:        /etc/ft-nixpalette/colors.json
                /etc/ft-nixpalette/themes.json
  Home Manager: $XDG_DATA_HOME/ft-nixpalette/colors.json
                $XDG_DATA_HOME/ft-nixpalette/themes.json

colors.json structure:
  { "themeId": "builtin:base/catppuccin-latte", "polarity": "light", "base16": { "base00": "eff1f5", ... } }

themes.json structure:
  { "<themeId>": { "themeId": "...", "polarity": "...", "base16": { ... }, "meta": { ... } }, ... }

All base16 hex values are plain 6-character strings WITHOUT a leading #.
Standard semantic mapping: base00=background, base05=foreground, base08=red,
base0B=green, base0D=blue, base0E=purple.
The `polarity` field ("dark"/"light") maps directly to GNOME's color-scheme setting.

## What ft-nixpalette-gnome should provide

A separate flake with nixosModules.default and homeModules.default that
adds GNOME theming on top of ft-nixpalette. All options live under the
`ft-nixpalette-gnome` namespace.

GNOME theming is done primarily via dconf settings, exposed in Home Manager as
dconf.settings or via programs.dconf. Use Home Manager's dconf.settings approach
throughout. Avoid writing raw gsettings commands in activation scripts.

### 1. Wallpaper via dconf
- Declare `ft-nixpalette-gnome.wallpaper.enable` (default: true)
- Set dconf keys:
    /org/gnome/desktop/background/picture-uri        → "file://${wallpaper}"
    /org/gnome/desktop/background/picture-uri-dark   → "file://${wallpaper}"
    /org/gnome/desktop/screensaver/picture-uri       → "file://${wallpaper}"
- Wallpaper comes from colors.json active theme or ft-nixpalette.defaultWallpaper

### 2. Color scheme (dark/light mode)
- Declare `ft-nixpalette-gnome.colorScheme.enable` (default: true)
- Map ft-nixpalette polarity to GNOME color-scheme:
    "dark"  → dconf /org/gnome/desktop/interface/color-scheme = "prefer-dark"
    "light" → dconf /org/gnome/desktop/interface/color-scheme = "prefer-light"
- Also set /org/gnome/desktop/interface/gtk-theme accordingly:
    dark  → "adw-gtk3-dark" (if available) else "Adwaita-dark"
    light → "adw-gtk3"      (if available) else "Adwaita"

### 3. GNOME Terminal / text-editor colors
- Declare `ft-nixpalette-gnome.terminal.enable` (default: true)
- Generate a GNOME Terminal profile via dconf using base16 colors:
    background-color → #base00
    foreground-color → #base05
    palette (16 ANSI colors mapped from base16 in standard order)
    bold-color       → #base07
    use-theme-colors → false

### 4. Extension: Night Theme Switcher compatibility
- Declare `ft-nixpalette-gnome.nightThemeSwitcher.enable` (default: false)
- If enabled, configure the Night Theme Switcher GNOME extension via dconf
  to use the correct GTK themes for day/light and night/dark modes
- This allows polarity-aware automatic switching

### 5. GTK colors via css override (for Adwaita)
- Declare `ft-nixpalette-gnome.gtkColors.enable` (default: false)
- Generate a ~/.config/gtk-4.0/gtk.css override injecting base16 as CSS variables:
    @define-color accent_color #<base0D>;
    @define-color accent_bg_color #<base0D>;
    @define-color accent_fg_color #<base00>;
    @define-color window_bg_color #<base00>;
    @define-color window_fg_color #<base05>;
    @define-color view_bg_color #<base01>;
    @define-color view_fg_color #<base05>;
    @define-color headerbar_bg_color #<base01>;
- Also write ~/.config/gtk-3.0/gtk.css with equivalent GTK3 variable names

### 6. Live theme switcher
- Provide package `ft-nixpalette-gnome.packages.${system}.ft-nixpalette-switch`
- Script reads themes.json, presents a zenity or yad picker (GUI-friendly),
  then on selection:
    1. Updates dconf wallpaper keys (gsettings set ...)
    2. Updates dconf color-scheme
    3. Writes ~/.cache/ft-nixpalette/active.json
    4. Optionally reloads GNOME Shell (gdbus call to reload)
- Works only for themes in themes.json (ft-nixpalette.preloadThemes)

## Architecture requirements

- flake input: inputs."ft-nixpalette".url = "github:FT-nixforge/ft-nixpalette"
- Re-export ft-nixpalette's modules
- Do NOT re-declare ft-nixpalette options
- GNOME-specific: many settings require a logout/login to fully apply (GTK theme,
  some dconf keys). Document this clearly. The switcher handles what it can at runtime.
- Home Manager module handles all dconf, GTK CSS, and the switcher script
- NixOS module ensures adw-gtk3 is installed system-wide if colorScheme.enable

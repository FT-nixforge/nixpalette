You are building `nixpalette-cosmic` — a NixOS/Home Manager flake that integrates
the nixpalette theme framework with the COSMIC desktop environment by System76.

## What nixpalette provides (the dependency)

nixpalette (github:FTMahringer/nixpalette) is a theme management flake built on
top of Stylix. At build time it writes two files:

  NixOS:        /etc/nixpalette/colors.json
                /etc/nixpalette/themes.json
  Home Manager: $XDG_DATA_HOME/nixpalette/colors.json
                $XDG_DATA_HOME/nixpalette/themes.json

colors.json structure:
  { "themeId": "builtin:base/dracula", "polarity": "dark", "base16": { "base00": "282a36", ... } }

themes.json structure:
  { "<themeId>": { "themeId": "...", "polarity": "...", "base16": { ... }, "meta": { ... } }, ... }

All base16 hex values are plain 6-character strings WITHOUT a leading #.
Standard semantic mapping: base00=background, base05=foreground, base08=red,
base0B=green, base0D=blue, base0E=purple.
`polarity` ("dark"/"light") maps directly to COSMIC's dark/light mode toggle.

## Context: COSMIC desktop

COSMIC (by System76) is a new Rust-based Wayland compositor and desktop environment,
currently in alpha/beta as of 2024-2025. It uses its own compositor (cosmic-comp),
panel (cosmic-panel), launcher (cosmic-launcher), and settings daemon. NixOS support
is provided by nixos-cosmic (github:lilyinstarlight/nixos-cosmic).

COSMIC stores its settings in RON (Rust Object Notation) config files under
~/.config/cosmic/. Direct file manipulation is the primary integration path
until a full Home Manager module is available.

## What nixpalette-cosmic should provide

A separate flake with nixosModules.default and homeModules.default that
adds COSMIC theming on top of nixpalette. All options live under the
`nixpalette-cosmic` namespace.

### 1. COSMIC theme mode (dark/light)
- Declare `nixpalette-cosmic.themeMode.enable` (default: true)
- Write ~/.config/cosmic/com.system76.CosmicTheme.Mode/v1 with:
    dark  → is_dark: true
    light → is_dark: false
- This is a plain RON file

### 2. COSMIC accent color
- Declare `nixpalette-cosmic.accentColor.enable` (default: true)
- Write the COSMIC accent color config from base0D (blue):
    ~/.config/cosmic/com.system76.CosmicTheme.Dark/v1
    or ~/.config/cosmic/com.system76.CosmicTheme.Light/v1
- The COSMIC theme file is a RON attrset with accent, bg, text colors
- Map base16 to COSMIC theme slots:
    accent       → base0D
    bg           → base00
    primary_container_bg → base01
    secondary_container_bg → base02
    on_bg        → base05
    on_bg_component → base04
    positive     → base0B (green)
    caution      → base0A (yellow)
    destructive  → base08 (red)

### 3. Wallpaper via cosmic-bg
- Declare `nixpalette-cosmic.wallpaper.enable` (default: true)
- Write ~/.config/cosmic/com.system76.CosmicBackground/v1 RON config:
    same-on-all → true
    wallpapers  → [("<output>", "<path>")]
- Wallpaper path from nixpalette.defaultWallpaper or active theme

### 4. COSMIC Terminal colors
- Declare `nixpalette-cosmic.terminal.enable` (default: true)
- Write ~/.config/cosmic/com.system76.CosmicTerm/v1 color palette:
    Map base16 to the 16 ANSI terminal colors in standard order
    Set background → base00, foreground → base05

### 5. Live theme switcher
- Provide package `nixpalette-cosmic.packages.${system}.nixpalette-switch`
- Script reads themes.json, presents fzf picker, then on selection:
    1. Writes new COSMIC theme RON files
    2. Signals cosmic-settings-daemon to reload (dbus or SIGHUP)
    3. Writes ~/.cache/nixpalette/active.json
    4. Sends notify-send confirmation
- Note: COSMIC is in active development; the config format may change.
  Document the COSMIC version this was tested against.

## Architecture requirements

- flake inputs:
    inputs.nixpalette.url    = "github:FTMahringer/nixpalette"
    inputs.nixos-cosmic.url  = "github:lilyinstarlight/nixos-cosmic"
- Re-export nixpalette's modules and nixos-cosmic's NixOS module
- Do NOT re-declare nixpalette options
- All config file writing is done via Home Manager's home.file mechanism
  (home.file."<path>".text = toRON ...)
- Implement a toRON Nix helper function or use builtins.toJSON with post-processing
  for the subset of RON that COSMIC actually uses (it's close to JSON for simple configs)
